Week 3 at Lambda School

Binding, Constructors, String Template Literals nad Protoypes

    Binding

  • Implicit binding uses the `this` keyboard to bind to an object. This is typically the object to the left of the .
  • Every function in Javascript has call, apply and bind from the object prototype
  • Objects are implicitly bound while functions are explicit. DOM will use explicit binding
  • The difference between .call and .apply is .call accepts an arguement list while .apply accepts an array
  • The difference between .bind and .call is that .bind stores the information for later which is typically assigned a variable. Both .bind and .call accepting an argument list
  • The spread operator or spread syntax on MDN uses ''...'' which renders apply useless. It allows the use of .call and .bind
  • Babel allows developers to write new ES6 code which will render browser compatible Javascript

    • Constructors

  • Constructors functions that are used to to build objects. They're basically object factories.
  • Best practice for consturctor functions are to name them with a capital letter e.g. Person
  • The new keyboard applies the this binding to a constructor function and lets the Javascript interpretter know that the function is a going to create a new object.
  • A constructor uses semicolons ; while objects are defined using commas ,

    • String Template Literals

  • String template literal use backticks `` for string interpolation e.g. console.log(`Hello ${this.name}`);

    • Prototypes

  • Methods are functions incide of an object
  • Javascript is a prototype based language rather than a class based language
  • Everything is an object in Javascript due to the dunder proto. Dunder refers to a double underscore.
  • Binding the this keyboard can be applied in a child constructor
    function Child(childAttributes) { Parent.call(this, childAttributes); }
  • Protohype functions are not automatically from child objects unless explicitly stated
    Child.protohype = Object.create(Parent.prototype);
  • New methods must be declared after the Object.create() constructor
